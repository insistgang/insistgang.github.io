2026年2月15日 星期日

昨日计划完成情况

  • 继续优化爬虫项目
  • 整理知识库和项目文件
  • 早睡早起（今日11点起床，未完成）

今日记录

上午

11点左右起床，送压岁钱。

下午

直接把 Kimi 的额度都用完了。对于昨天做的一些项目还有自己以前做的一些项目，我做了一下 dashboard，非常有意思的可视化，包括：

  • 论文阅读数据采集
  • 竞赛的分配
  • 整个电脑磁盘的文件重新归档

一下子把 Kimi 的 API 额度用完了。然后发现如果想用 GLM-5 的话还是非常困难，没有把 Agent 用起来，每次最多用 50% 的余额就刷新了。

问了 Claude 之后发现一个新的观点：GLM 涨价了，大部分人付费的只有 20% 的人用的比例算是比较高的，而 80% 的人仅用于聊天和简单的操作，所以他们的消耗很小。由于 OpenClaw 以及 Opus 4.6 集群出来之后，大部分人都能拉到 100%，所以 GLM 就产生了提价这个操作。

但是我还是没有能够拉满。Claude 给我的反馈很经典：那 20% 的核心特征不是”聊得多”，而是把 token 转化成了可交付的成果。这个很关键，我要把它的 token 转化为我自己的核心结果。

晚上

姐夫过生日，一起去酒店给他庆生，喝了一些红酒，吃了很多熟食。

下午去洗了个澡，晚上和爸爸妈妈去超市购物了，买了一些零食和饮料。晚上和好久没聊的朋友唠了会嗑。

加急用了一下 GLM，之前祖师爷的论文下载了很多存在问题，用了 Open Code 才发现，又重新整理下载新的论文，继续重新做了整合。

状态

精力 6/10，情绪 6/10

健康打卡

  • 午餐：青菜煮肉、羊肉汤、蒜苔肉丝
  • 晚餐：酒店庆生宴（红酒、熟食）
  • 运动：无
  • 喝水：2L

今日收获

  • 发现了 AI 工具使用的核心：把 token 转化成可交付的成果，而不是仅仅聊天
  • 用 Kimi 制作了项目 dashboard 可视化，效果不错
  • 重新整理了祖师爷的论文资源

今日卡点

  • GLM-5 的 Agent 模式还是没有完全用起来，每次只能用到 50% 额度就刷新
  • 11 点起床，早睡早起计划未完成
  • 没有运动

感悟

大模型时代的核心用户（20%）的特征不是”用得多”，而是把 token 转化成了可交付的成果。这是对工具价值最大化的关键认知。
