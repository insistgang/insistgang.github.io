# 第21周复盘（5月18日 ~ 5月24日）

## 本周总结

本周是**多线并行、论文持续推进**的一周。论文实现连续7天推进的突破，逢甲赛持续迭代，工具链持续升级。但六级学习连续5天未完成是本周最大的失败，早睡习惯也一直未建立。

## 关键成就

1. **论文连续7天推进** - 从5/21启动论文转投，到5/25的visio配图，实现连续7天推进的突破 🎉
2. **逢甲赛持续迭代** - 修复志愿者模块、接入mimo+codex、修复CI-CD bug
3. **工具链升级** - 建立论文阅读系统、学习5篇论文、搭建聚合平台和知识库平台
4. **软著提交** - 完成5个软著短信接收和材料提交
5. **竞赛讲座** - 给研一同学做竞赛讲座，输出倒逼输入
6. **心理咨询** - 完成咨询并进行自我觉察反思
7. **运动保持** - 游泳3天，保持运动习惯

- 情绪低谷：周二（5/10）- Claude宕机、任务切换频繁

- **规律**：早起+黄金时段聚焦+晚上运动 = 高精力日

### 成功模式

**本周成功的任务有什么共同点？**
- 论文推进：有明确的"下一步"（改论文、画图、投稿）
- 游泳运动：有固定的触发时间（晚上）
- 工具升级：有即时反馈（安装后立即使用）

**在什么时间段效率最高？**
- 上午9:00-12:00（黄金时段）：适合论文、实验
- 晚上20:00-23:00（安静时段）：适合学习、工具升级

**使用了什么工具/方法？**
- Claude Code + OpenCode（工具链）
- draw.io mcp（论文绘图）
- visio（技术路线图）

### 待改进领域

**本周失败/未完成的任务有哪些？**
- 六级学习：连续5天未完成
- 早睡：连续7天未达标
- 任务并行：多线切换导致碎片化

**失败的原因是什么？**
- 六级：没有固定触发时间，被其他任务挤压，优先级最低
- 早睡：晚上安静时段想多做事，没有固定的"关闭时间"
- 任务并行：外部事件插入（导师通知、Claude宕机）导致被动响应

**如何调整或拆解？**
- 六级：绑定到早晨第一件事，最小启动（背10个单词）
- 早睡：设定23:30的"硬关闭"闹钟，到点必须关电脑
- 任务并行：减少同时推进的项目数量，聚焦1-2个核心
